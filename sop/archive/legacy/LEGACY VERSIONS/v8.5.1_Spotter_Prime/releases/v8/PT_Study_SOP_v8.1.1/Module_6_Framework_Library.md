## MECHANISM CHAINS (M1-M8)

### M1: Input → Process → Output → Consequence

**Universal mechanism chain for any system.**

| Step | Question |
|------|----------|
| Input | What enters the system? |
| Process | What does the system do to it? |
| Output | What leaves the system? |
| Consequence | What changes because of it? |

**Best for:** Physiology, pathology, biomechanics, reasoning  
**Example:** Glucose (input) → Glycolysis (process) → ATP (output) → Muscle contraction (consequence)

---

### M2: Trigger → Mechanism → Result → Implication

**Cause-effect chain with meaning.**

| Step | Question |
|------|----------|
| Trigger | What initiates the change? |
| Mechanism | What process does the system run? |
| Result | What is produced/changed? |
| Implication | Why does it matter? |

**Best for:** Injury, physiology, pharmacology, learning, adaptation  
**Example:** Tissue overload (trigger) → Inflammatory cascade (mechanism) → Swelling/pain (result) → Rest needed for healing (implication)

---

### M3: Load/Stress → System Response → Threshold → Outcome

**Demand vs. capacity model.**

| Step | Question |
|------|----------|
| Load/Stress | What force/demand is applied? |
| System Response | How does the system initially react? |
| Threshold | When does it break/plateau/adapt? |
| Outcome | What happens next? |

**Best for:** Tissues, metabolic systems, psychological stress, dosing, progression  
**Example:** Progressive exercise load → Muscle micro-damage → Adaptation threshold → Hypertrophy

---

### M4: Deficit → Compensation → Side Effects → Long-Term Pattern

**Workaround consequences.**

| Step | Question |
|------|----------|
| Deficit | What is missing? |
| Compensation | What workaround appears? |
| Side Effects | What new problems occur? |
| Pattern | What chronic strategy forms? |

**Best for:** Movement, cognition, metabolic, cardiorespiratory, motor control  
**Example:** Quad weakness (deficit) → Hip hike gait (compensation) → Low back pain (side effect) → Chronic altered gait pattern

---

### M5: Signal → Detection → Processing → Action

**Information flow.**

| Step | Question |
|------|----------|
| Signal | What information exists? |
| Detection | What system reads it? |
| Processing | How is it interpreted? |
| Action | What behavior/output follows? |

**Best for:** Neurophysiology, sensory systems, motor control, decision-making  
**Example:** Joint position (signal) → Proprioceptors (detection) → Cerebellum (processing) → Postural adjustment (action)

---

### M6: Perturbation → Stability Mechanism → Correction → Return to Baseline

**Homeostatic control.**

| Step | Question |
|------|----------|
| Perturbation | What pushed system off target? |
| Stability Mechanism | What restores balance? |
| Correction | How is error reduced? |
| Return to Baseline | When is normal achieved? |

**Best for:** Balance, homeostasis, control systems  
**Example:** Blood pressure drop (perturbation) → Baroreceptor reflex (stability) → Vasoconstriction + HR increase (correction) → BP normalized (baseline)

---

### M7: Resource → Utilization → Limiter → Efficiency

**Performance economics.**

| Step | Question |
|------|----------|
| Resource | What is available? |
| Utilization | How is it used? |
| Limiter | What restricts performance? |
| Efficiency | How well is resource converted to output? |

**Best for:** Metabolism, endurance, learning time, workload management  
**Example:** Glycogen stores (resource) → Aerobic metabolism (utilization) → Lactate threshold (limiter) → VO2max efficiency

---

### M8: Cause → Mechanism → Sign → Test → Confirmation

**Diagnostic reasoning chain.**

| Step | Question |
|------|----------|
| Cause | What started the problem? |
| Mechanism | What internal process failed? |
| Sign | What observable change appears? |
| Test | What verifies it? |
| Confirmation | What finding seals the diagnosis? |

**Best for:** Any diagnostic or evaluative system  
**Example:** ACL tear (cause) → Knee instability (mechanism) → Swelling + giving way (sign) → Lachman test (test) → Positive anterior translation (confirmation)

## HIERARCHY SYSTEMS (H1-H8)

### H1: System → Subsystem → Component → Element → Cue

**Master framework for organizing any domain.**

| Level | Definition |
|-------|------------|
| System | Big topic |
| Subsystem | Major subdivision |
| Component | Parts of each subsystem |
| Element | Smallest functional unit |
| Cue | Identifiable behavior/sign/function |

**Best for:** Any domain mapping  
**Example:** Nervous system → Peripheral nervous system → Brachial plexus → Musculocutaneous nerve → Biceps reflex

---

### H2: Structure → Function → Behavior → Outcome

**Form to result.**

| Level | Definition |
|-------|------------|
| Structure | What it is |
| Function | What it does |
| Behavior | How it acts dynamically |
| Outcome | What results in the real world |

**Best for:** Any biological or engineered system  
**Example:** Biceps brachii (structure) → Elbow flexion (function) → Carries load to mouth (behavior) → Eating, lifting (outcome)

---

### H3: Macro → Meso → Micro

**Scale levels.**

| Level | Definition |
|-------|------------|
| Macro | Whole system |
| Meso | Organizational layer |
| Micro | Smallest active unit |

**Best for:** Anatomy, physiology, pathology, biomechanics, learning, rehab  
**Example:** Respiratory system (macro) → Lungs/airways (meso) → Alveoli/capillaries (micro)

---

### H4: Concept → Sub-Concept → Example → Application

**Abstract to concrete.**

| Level | Definition |
|-------|------------|
| Concept | Main idea |
| Sub-Concept | Core branches |
| Example | Concrete instantiation |
| Application | How it is used in context |

**Best for:** Any class where concepts must be learned abstractly  
**Example:** Motor learning (concept) → Stages of learning (sub-concept) → Cognitive stage in new patient (example) → High feedback, simple tasks (application)

---

### H5: Principle → Rule → Procedure → Action Step

**Theory to action.**

| Level | Definition |
|-------|------------|
| Principle | Governing idea |
| Rule | Constraints or conditions |
| Procedure | Series of steps |
| Action Step | The user's actual move |

**Best for:** Labs, interventions, protocol learning  
**Example:** Sterile technique (principle) → Never touch sterile with non-sterile (rule) → Wound dressing procedure (procedure) → Apply gauze without contamination (action)

---

### H6: Theory → Model → Mechanism → Prediction

**Explanation to forecast.**

| Level | Definition |
|-------|------------|
| Theory | Guiding explanation |
| Model | Representation of system |
| Mechanism | Operation inside the model |
| Prediction | What should happen |

**Best for:** Physiology, biomechanics, neuroscience, pathology  
**Example:** Pain gate theory (theory) → Spinal gating model (model) → Large fiber activation inhibits transmission (mechanism) → TENS should reduce pain (prediction)

---

### H7: Goal → Requirement → Method → Metric

**Outcome planning.**

| Level | Definition |
|-------|------------|
| Goal | What outcome you want |
| Requirement | Conditions needed |
| Method | How you achieve it |
| Metric | How you know it worked |

**Best for:** Interventions, clinical decision-making, study planning  
**Example:** Improve gait speed (goal) → Adequate quad strength (requirement) → Progressive resistance exercise (method) → 10m walk test improvement (metric)

---

### H8: Agent → Interaction → Change → Effect

**Actor-driven change.**

| Level | Definition |
|-------|------------|
| Agent | Actor influencing system |
| Interaction | How they meet the system |
| Change | What shifts internally |
| Effect | External result |

**Best for:** Drugs, tissues, devices, behavioral factors  
**Example:** NSAIDs (agent) → COX enzyme inhibition (interaction) → Reduced prostaglandins (change) → Decreased inflammation/pain (effect)

## HYBRID FRAMEWORKS (Y1-Y5)

### Y1: What It Is → What It Does → How It Fails → What That Looks Like

**Universal diagnostic/understanding lens.**

| Step | Focus |
|------|-------|
| What it Is | Identity/structure |
| What it Does | Normal function |
| How it Fails | Dysfunction mechanism |
| What That Looks Like | Observable signs/symptoms |

**Best for:** Understanding any concept  
**Example:** ACL (what it is) → Prevents anterior tibial translation (what it does) → Ruptures under valgus/rotation force (how it fails) → Swelling, instability, positive Lachman (looks like)

---

### Y2: Normal → Perturbed → Compensated → Decompensated

**Universal progression model.**

| Stage | Description |
|-------|-------------|
| Normal | Baseline healthy state |
| Perturbed | Initial insult/change |
| Compensated | System adapts, function maintained |
| Decompensated | Adaptation fails, dysfunction emerges |

**Best for:** Disease, dysfunction progression  
**Example:** Normal heart → Valve disease (perturbed) → LV hypertrophy maintains output (compensated) → Heart failure symptoms (decompensated)

---

### Y3: Baseline State → Input → Transformation → Output → Next State

**Universal systems model.**

| Step | Description |
|------|-------------|
| Baseline State | Starting condition |
| Input | What enters |
| Transformation | Process applied |
| Output | What emerges |
| Next State | New baseline |

**Best for:** Any process  
**Example:** Resting muscle (baseline) → Neural signal (input) → Excitation-contraction coupling (transformation) → Force production (output) → Contracted state (next state)

---

### Y4: Capability → Limitation → Adaptation → Optimization

**Performance model.**

| Step | Description |
|------|-------------|
| Capability | What the system can do |
| Limitation | Constraints on performance |
| Adaptation | How system adjusts |
| Optimization | Best achievable outcome |

**Best for:** Rehab, motor learning, performance  
**Example:** Shoulder flexion 90° (capability) → Pain limits above 60° (limitation) → Modify exercise below pain (adaptation) → Gradual ROM increase (optimization)

---

### Y5: Information → Integration → Decision → Execution

**Cognitive-motor chain.**

| Step | Description |
|------|-------------|
| Information | Sensory input received |
| Integration | Processing and combining |
| Decision | Selection of response |
| Execution | Motor/behavioral output |

**Best for:** Neuro, cognition, behavior, movement  
**Example:** See ball approaching (information) → Calculate trajectory + timing (integration) → Decide to catch (decision) → Reach and grasp (execution)
